# コミットメッセージ規約

## フォーマット

```

<種類>: <内容>

```

## 種類（プレフィックス）

- **feat** → 新しい機能の追加
  - 例: `feat: ユーザー登録機能を追加`
- **fix** → バグ修正
  - 例: `fix: ログインエラーを修正`
- **docs** → ドキュメントの変更
  - 例: `docs: README を更新`
- **refactor** → リファクタリング
  - 例: `refactor: ディレクトリ構造の変更`
- **chore** → その他の変更
  - 例: `chore: その他の変更`

## ルール

1. 種類は必ず先頭につける
2. 内容は「何をしたか」を簡潔に
3. 日本語 OK（英語でも可）
4. 1 行目は 50 文字以内を目安に

## 例

```bash
git commit -m "docs: GitHub Flow についてのメモを追加"
```

対応する絵文字を決めることで、視覚的に見やすくすることも可能です！

例：

```bash
git commit -m "📝 docs: GitHub Flow についてのメモを追加"
```
